SIMILARITY MATRIX FILE FORMAT

The similarity matrix file is a plain .csv (comma-separated) or .tsv (tab-separated) file. Values should be in the range 0–1 (higher = more similar). Values outside this range are auto-normalised.

MINIMAL FORMAT (no header, N×N matrix, classes assumed 1…N):
1.000,0.823,0.341,0.102
0.823,1.000,0.512,0.089
0.341,0.512,1.000,0.774
0.102,0.089,0.774,1.000
WITH CLASS NUMBER HEADER (first row = class numbers, first col = class numbers):
class,1,3,7,12
1,    1.000,0.823,0.341,0.102
3,    0.823,1.000,0.512,0.089
7,    0.341,0.512,1.000,0.774
12,   0.102,0.089,0.774,1.000

Class numbers in the header/row-labels do not need to be contiguous or start at 1 — they map directly to the RELION class numbers in your _model.star.

MULTIPLE MATRICES IN ONE FILE (separated by # name section headers):
# Cross-Correlation
class,1,3,7,12
1,1.000,0.823,0.341,0.102
3,0.823,1.000,0.512,0.089
7,0.341,0.512,1.000,0.774
12,0.102,0.089,0.774,1.000

# Euler Angle Distance
class,1,3,7,12
1,1.000,0.650,0.200,0.050
3,0.650,1.000,0.430,0.110
7,0.200,0.430,1.000,0.880
12,0.050,0.110,0.880,1.000

Each # Name line starts a new matrix block. Matrices are separated by a blank line. You can include as many matrices as you like — switch between them in the sidebar.

Tip: The matrix does not need to cover all classes — only classes present in both the matrix and your loaded _model.star will show suggestions. The diagonal (self-similarity) is always ignored.
